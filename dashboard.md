# Golf Tracker Dashboard Implementation Plan

본 문서는 골프 라운드 기록을 시각화하여 한눈에 볼 수 있는 대시보드 구현 계획을 담고 있습니다. 사용자 요청에 따라 모바일과 PC 환경에 최적화된 서로 다른 뷰를 제공합니다.

## 1. UI 수정: 홈 메뉴 (app/page.js)
- **위치**: 각 라운드 항목의 '클라우드 업로드(☁️)' 버튼 좌측
- **요소**: 'D' 라벨이 붙은 버튼 추가
- **동작**: 클릭 시 해당 라운드의 ID를 가지고 `/dashboard?id=[roundId]` 페이지로 이동

## 2. 대시보드 페이지 설계 (app/dashboard/page.js)

### 2.1 공통 기능
- **데이터 로딩**: `localStorage` 또는 API를 통해 라운드 상세 데이터(holes, shots 등)를 가져옴
- **반응형 대응**: CSS Media Queries를 사용하여 화면 너비에 따라 PC 버전과 모바일 버전을 스위칭

### 2.2 PC 버전 (상세 시트 뷰)
PC 환경에서는 엑셀/스프레드시트와 유사한 고밀도 정보 뷰를 제공합니다.

#### A. 상단 스코어카드 테이블
- **행 구성**: Hole, Par, Score, Putt, T-Club, F/W, GIR, Penalty, Sand Save, MEMO
- **열 구성**: 1~9홀, 전반합계, 10~18홀, 후반합계, 전체합계
- **스타일링**: 격자 무늬 테이블, 성적에 따른 배경색(버디 이하, 보기 이상 등)

#### B. 중앙 샷 상세 그리드
- 각 홀별로 기록된 모든 샷을 수직으로 나열
- **표시 항목**: Club, Shot, Landing, Distance, T.Dis, R.Dis, Penalty, MEMO
- 한 화면에 18홀의 모든 샷 데이터를 그리드 형태로 배치

#### C. 우측 통계 패널
- **클럽별 통계**: 사용 횟수, 해저드/OB 기록, 평균 거리
- **스코어 분석**: 3퍼트 횟수, 드라이버 비거리, 버디/파/보기/더블/트리플 등 개수 요약
- **실수 분석**: 탑볼, 뒷땅, 생크 등 메모 기반 통계
- **종합 메모**: 라운드 총평 표시

### 2.3 모바일 버전 (간편 요약 뷰)
모바일 환경에서는 세로로 긴 형태의 카드 UI를 제공합니다.

#### A. 상단 주요 지표 (Stats Cards)
- 스코어 (오버파 표시), GIR %, 평균 퍼트 수, 총 비거리 등을 큰 글씨의 카드로 배치

#### B. 코스 요약 테이블
- 전/후반을 나누어 Hole, Par, Score, Putt, Penalty, GIR 등을 요약한 테이블


#### C. 하단 메모
- 라운드 전체 메모를 읽기 쉬운 형태로 배치

## 3. 기술적 세부 사항

### 데이터 매핑 규칙
- **GIR (Green In Regulation)**: 파4에서 2온 이하, 파5에서 3온 이하, 파3에서 1온 달성 여부 계산
- **F/W (Fairway Hit)**: 티샷(첫 번째 샷)의 Landing이 'F'인 경우
- **Score**: `shots.length` + 벌타(Penalty 'O'=2, 'H'=1 등) 계산
- **Putt**: `club === 'Pt'` 인 샷의 개수

### 디자인 시스템
- 기존 앱의 **Dark Mode** 및 **Glassmorphism** 스타일 유지
- 네온 컬러(Accent Neon)를 활용하여 주요 지표 강조
- PC 버전의 경우 `overflow-x: auto`를 사용하여 넓은 테이블 대응

## 4. 향후 작업 순서
1. `app/dashboard/page.js` 신규 파일 생성 및 기본 레이아웃 구성
2. PC 버전용 대형 테이블 컴포넌트 구현
3. 모바일 버전용 통계 카드 및 요약 뷰 구현
4. `app/page.js`에 진입 버튼 추가 및 라우팅 연결

# 핀 홀 및 퍼팅 GPS 위치 저장 기능 구현 계획

이 문서는 `Ho`(핀 홀 위치) 및 `Pt`(퍼팅 위치) 버튼을 눌렀을 때 GPS 좌표를 획득하고 저장하는 기능에 대한 기술 검토 및 구현 계획을 담고 있습니다.

## 1. 기술 검토: 모바일 크롬 위치 정보 획득

### 가능 여부
- **모바일 크롬(Chrome for Android/iOS)**은 W3C **Geolocation API**를 완벽하게 지원합니다.
- `navigator.geolocation.getCurrentPosition()`을 사용하여 현재 위도(latitude)와 경도(longitude)를 획득할 수 있습니다.

### 주요 요구 사항 및 제약
- **HTTPS 필수**: 최신 브라우저는 보안상의 이유로 HTTPS 환경에서만 위치 정보 접근을 허용합니다.
- **사용자 권한**: 브라우저에서 위치 정보 접근 권한 팝업이 뜰 때 사용자가 '허용'을 눌러야 합니다.
- **정확도**: 실외(골프장)에서는 GPS 위성을 통해 상당히 높은 정확도(수 미터 이내)를 얻을 수 있으나, 첫 호출 시 응답 속도가 약간 느릴 수 있습니다. (`enableHighAccuracy: true` 옵션 권장)

## 2. 데이터 구조 변경 제안

### Hole 데이터 (핀 위치 저장용)
```javascript
{
  hole: 1,
  par: 4,
  // ... 기존 필드
  pinCoords: { lat: 37.xxx, lng: 127.xxx, timestamp: "..." } // 추가
}
```

### Shot 데이터 (퍼팅 위치 저장용)
```javascript
{
  id: "...",
  club: "Pt",
  // ... 기존 필드
  coords: { lat: 37.xxx, lng: 127.xxx } // 추가
}
```

## 3. 기능 구현 계획

### A. Ho (Hole) 버튼 기능
- **위치**: 그린 프리뷰 창 근처 또는 네비게이션 영역에 배치.
- **동작**:
  1. 버튼 클릭 시 `navigator.geolocation` 호출.
  2. 위치 정보를 받으면 현재 홀의 `pinCoords` 필드에 저장.
  3. 성공 알림 표시 (예: "홀 위치가 저장되었습니다.")

### B. Pt (Putt) 버튼 기능 (퀵 등록)
- **위치**: 현재 `SHOT+` 버튼 근처 또는 하단 퀵 메뉴.
- **동작**:
  1. 버튼 클릭 시 즉시 위치 정보 획득 시작.
  2. 위치 정보를 받으면 `club: 'Pt'`인 새로운 샷 객체를 생성하여 현재 홀의 `shots` 배열에 추가.
  3. 위치 정보(`coords`)를 포함하여 1차 등록 완료.
  4. 나중에 상세 내용(컨시드 여부 등)을 수정할 수 있도록 기존 샷 수정 모달과 연동.

### C. 공통 유틸리티 구현
- `utils/location.js` (가칭) 생성:
  - 위치 정보를 Promise 형태로 반환하는 래퍼 함수 구현.
  - 타임아웃 및 에러 처리 로직 포함.

## 4. 향후 확장성
- 저장된 GPS 좌표를 바탕으로 **퍼팅 거리(핀-퍼터 간 거리) 자동 계산** 기능 추가 가능.
- 라운드 종료 후 전체 샷의 궤적을 지도 위에 시각화.
